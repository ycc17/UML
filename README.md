# **詳解UML類別圖&物件圖**
## UML 類別圖
描述類別、介面、協作及他們之間的關係的圖。顯示系統中類別的靜態結構。
## 有什麼作用？
描述軟體系統的靜態結構- 對系統的詞彙建模- 對簡單協作建模- 對邏輯資料庫模式建模
## 類別圖說明
- 類別名為斜體則為抽象類別類別
- 方法為斜體則為抽象方法

1. 第一行：類別名稱
2. 第二行：類別屬性
3. 第三行：類別方法

- 類別/屬性/方法說明：
1. '-' 表示私有(private)
2. ' #'表示保護(protected)
3. '+'表示公開(public)
